### 详细元素说明

#### 顶行标签

| 标签 | 含义 | 颜色 | 示例 |
|------|------|------|------|
| 活跃度 | 仓库活跃程度 | 根据 level | `[🔥 今天活跃 (0d)]` |
| 语言 | 编程语言 | info(蓝) | `[Java]` `[Python]` |
| 归档 | 归档状态 | warning(黄) | `[归档]` (if archived) |
| Fork | fork 标记 | info(蓝) | `[Fork]` (if fork) |
| 同步状态 | 克隆/同步状态 | 多种 | 见下表 |

#### 同步状态标签

| 进行中状态 | 持久状态 | 颜色 | 说明 |
|----------|--------|------|------|
| `[排队中]` (queued) | - | info(蓝) | 等待同步 |
| `[同步中]` (syncing) | - | warning(黄) | 正在同步 |
| `[完成]` (done) | - | success(绿) | 本次同步完成 |
| `[失败]` (failed) | - | danger(红) | 本次同步失败 |
| - | `[已同步]` | success(绿) | 已同步状态 |
| - | `[未同步]` | danger(红) | 未同步状态 |

#### 底部信息行

```
最后推送：2026-07-16  ·  默认分支：main
└─ 格式：YYYY-MM-DD 格式的日期，来自 pushed_at
└─ 用途：判断是否需要重新同步

最后同步：2 小时前  ·  ★ 0
├─ 相对时间格式：
│  ├─ 刚刚 (< 1 分钟)
│  ├─ N 分钟前 (< 60 分钟)
│  ├─ N 小时前 (< 24 小时)
│  ├─ N 天前 (< 7 天)
│  └─ MMM DD HH:mm (>= 7 天)
├─ 如果未同步过显示：未同步过
└─ ★ N 显示 star 数（仅当 > 0 时）
```

## ⚡ 按钮交互详解

### 三个主要按钮

#### 1. [重新同步] / [同步]
```
点击
  ↓
POST /api/repos/sync
  body: ["WeFi-HLB/apigw"]
  ↓
响应: { queued: ["WeFi-HLB/apigw"] }
  ↓
创建同步任务记录
syncJobs["WeFi-HLB/apigw"] = { status: "queued", error: null }
  ↓
启动轮询 (2秒一次)
  ↓
显示同步进度卡片
  ↓
完成后：
  - 刷新 repos 列表
  - 更新 is_cloned, branch_count, total_commits 等
  - 展示贡献者列表
```

#### 2. [复制命令]
```
点击
  ↓
GET /settings/proxy (获取代理配置)
  ↓
构建 git 命令：

git \
  -c core.compression=0 \                    // 禁用压缩（加速）
  -c http.postBuffer=524288000 \             // 500MB 缓冲
  -c http.lowSpeedLimit=0 \                  // 无速度限制
  -c http.lowSpeedTime=999999 \              // 11.6 天超时
  -c [https|http].proxy=http://127.0.0.1:7897 \  // 代理（可选）
  clone --mirror <clone_url> ./.data/repos/<full_name>

  或

  -C ./.data/repos/<full_name> fetch --all --prune
  ↓
navigator.clipboard.writeText(command)
  ↓
显示成功提示
```

#### 3. [展开] / [收起]
```
点击
  ↓
toggle(repo.full_name)
  ↓
expanded.has(repo.full_name) ? remove : add
  ↓
el-collapse-transition 动画
  ↓
展开贡献者表格
```

## 🔍 搜索和分组逻辑

### 搜索范围
```
用户输入：[搜索仓库名...]

搜索范围：
├─ 仓库全名 (full_name)
│  └─ 例：WeFi-HLB/apigw
└─ 仓库描述 (description)
   └─ 例：API Gateway Service

搜索规则：
├─ 不区分大小写
├─ 支持部分匹配
└─ 例：输入 "api" 会匹配 "WeFi-HLB/apigw" 和 "API Gateway"
```

### 分组规则
```
分组字段：activity.level（由后端返回）

分组顺序：
1️⃣ today         - 🔥 今天活跃
2️⃣ this_week    - 📈 本周活跃
3️⃣ this_month   - ⚙️ 本月活跃
4️⃣ pending      - ⏳ 待同步

分组表头格式：
🔥 今天活跃 (3 个)

含义：
├─ 分组标签（图标 + 文本）
├─ 仓库计数
└─ 如果计数为 0，该分组不显示
```

## 🎬 完整使用流程示例

### 场景 1: 查看仓库详情

```
1. 打开仓库看板
   ↓ 页面加载
   ↓ 显示 56 个仓库，按活跃度分组

2. 找仓库 "apigw"
   ├─ 搜索框输入 "apigw"
   ├─ 页面实时过滤
   └─ 只显示 WeFi-HLB/apigw

3. 查看仓库摘要
   ├─ 仓库名：WeFi-HLB/apigw
   ├─ 活跃度：⏳ 待同步 (3d)
   ├─ 编程语言：Java
   ├─ 最后推送：2026-07-16
   └─ 最后同步：2 小时前

4. 点击 [展开]
   ↓ 表格展开
   ↓ 显示 10 位贡献者

5. 查看贡献者
   ├─ 按提交数排序
   ├─ 张三：42 次提交
   ├─ 李四：28 次提交
   └─ ...
```

### 场景 2: 同步未同步仓库

```
1. 点击 [全量更新]
   ↓ 如果有未同步仓库，优先同步未同步的
   ↓ 否则，全量更新所有仓库

2. 同步进度卡片出现
   ├─ 显示 "同步进度"
   ├─ 每个仓库显示一行
   └─ 显示进度计数（3/56）

3. 实时更新
   ├─ 每 2 秒查询一次状态
   ├─ 仓库从 "排队" → "同步中" → "完成"
   └─ 如果失败，显示 "失败" 和错误信息

4. 全部完成
   ├─ 自动停止轮询
   ├─ 重新加载 repos 列表
   ├─ 更新 branch_count, total_commits 等
   └─ 页面刷新

5. 清除进度卡片
   ├─ 点击 [清除]
   └─ 进度卡片消失
```

### 场景 3: 手动同步仓库

```
1. 点击某仓库的 [复制命令]
   ↓ 命令复制到剪贴板

2. 命令内容（示例）：
   git -c core.compression=0 -c http.postBuffer=524288000 \
     -c http.lowSpeedLimit=0 -c http.lowSpeedTime=999999 \
     -c https.proxy=http://127.0.0.1:7897 \
     clone --mirror https://github.com/WeFi-HLB/apigw.git \
     ./.data/repos/WeFi-HLB/apigw

3. 在终端粘贴并执行
   └─ 本地手动同步仓库

4. 回到页面
   ├─ 点击 [重新同步]
   └─ 更新同步时间戳
```

## 📋 关键组件对应

| 功能 | 组件 | 实现 |
|------|------|------|
| **搜索** | el-input | v-model 双向绑定，computed 过滤 |
| **同步进度** | el-card + el-tag | 动态显示 syncJobs |
| **仓库卡片** | el-card | v-for 遍历 repos，shadow="hover" |
| **贡献者表** | el-table | :data="contributors", el-table-column 定义列 |
| **展开动画** | el-collapse-transition | 条件判断展开状态 |
| **按钮** | el-button | 不同类型和大小 |
| **标签** | el-tag | 不同颜色类型 |
| **图标** | el-icon | Loading, CircleCheck, CircleClose, Clock, DocumentCopy |
